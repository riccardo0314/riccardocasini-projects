# riccardocasini-projects
Personal projects build on my own for actual needs and I think are worth showcase


These are independent projects I built on my own, each born from a real problem rather
than as a polished showcase put together for recruiters. I'm including them because I
think they're worth showing: they reflect how I actually work 

Projects are deliberately different in nature. There's an exercise of reverse-engineering
and resilience against an undocumented, defensive API, a small distributed
system built for an industrial environment, under real hardware and OS constraints, and
running in production, and probably more in the future. 

They show the same thing from different angles:  start
from a practical problem and engineer my way to a working solution.
